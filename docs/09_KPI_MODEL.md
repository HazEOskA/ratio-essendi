# KPI Model

The main KPI is not number of agents.

The main KPI is value.

## Primary KPIs

The system exists to optimize:

- profit,
- client value,
- profitability,
- delivery speed,
- margin,
- useful output,
- operational leverage,
- system resilience.

## Leading KPIs

Profit is a late signal.

To control the system, earlier KPIs are required:

- offers created,
- offers sent,
- response rate,
- booked calls,
- demo bookings,
- conversion rate,
- delivery time,
- cost per lead,
- cost per task,
- margin per project,
- client satisfaction,
- number of reusable patterns,
- error rate,
- drift rate,
- failover recovery time.

## KPI Rule

Every agent must have at least one KPI.

Every cell must have KPIs.

Every strategy must define success and failure before execution.

No KPI means no activation.

## KPI Example

```txt
Sales Agent:
- response rate
- demo booked
- qualified leads
- cost per booked call

Product Agent:
- feature delivery time
- bug count
- validated feature count
- user value score

Finance Guard:
- budget usage
- burn rate
- margin
- blocked risky spend

Security Cell:
- blocked unsafe actions
- secret exposure incidents
- policy violations
- recovery time
```
